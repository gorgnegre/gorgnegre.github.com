Tree Command as a Sitemap Generator

Published • 22 Jul 2012 • tree • shell • web • sitemap

This short article shows a simple use of the venerable tree command to produce a clickable html sitemap of a static website for humans.

As I actually begin to populate this website, I am using tree as a quick hack to generate its sitemap. You can achieve better looks by playing a bit with CSS. 

[astro@gorgnegre:/var/www/rootdir/]% tree --dirsfirst -a \
-T "Sitemap for www.gorgnegre.com" -P "*html" \
-I "files|tasks" -C -H "http://www.gorgnegre.com" \
--charset UTF-8  -o files/pages/sitemap.html
  • This command is executed from the root directory of the web site. The options used are:
    --dirsfirst
    as the name implies, list directories before files on the same hierarchy
    -a
    list all, including hidden files.
    -T title
    sets the title and H1 header string (in HTML output mode).
    -P pattern
    list only those files that match the wild-card pattern. In our example above, I decided to list only the html files of my website so visitors can jump to an actual webpage when they click on an item of the sitemap.
    -I pattern
    do not list those files that match the wild-card pattern. In my example, I exclude all files in the files and tasks subdirectories in my webserver rootdir.
    -C
    Colorise output. Needed if you wish to use colors via CCS stylesheet.
    -H baseHREF
    turn on HTML output. baseHREF gives the base location.
    --charset
    set the character set to use.
    -o filename
    output to filename.
